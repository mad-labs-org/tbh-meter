**Global notes (no table row):**
- DAMAGE AND DPS ARE NEITHER LOGGED NOR STORED. No *Log carries damage; `DamageInfo` is transient; the 12 core stats and `b_isLive` are Obscured. **DPS meter core = sum of the Monsters' HP drops** (`pe@0x40` via `MonsterSpawnManager.MonsterList`/`DeadMonsterUnit`). Crit/element per hit not recoverable read-only.
- Run GOLD/XP/SOUL STONE/KILLS: two paths now. (a) DELTA of snapshots in the PLAIN SAVE: `CurrencySaveData.Quantity` (gold), `HeroSaveData.HeroExp` (xp), `aggregateSaveDatas`/inventory. (b) LIVE via `ut.beid` (PLAIN singleton) for kills/gold/box tick by tick — live mirror of the aggregate. The raw live wallets/inventories (`ud.su`/`LocalInventoryManager`) are obfuscated + Obscured (avoid).
- The entire SAVE is PLAINTEXT; Obscured/ACTk only in the RUNTIME mirrors (`ud.su`, `ud.th.te`, `uf`, 12 core stats, `b_isLive`). Useful exception: `ut.beid` is runtime PLAIN. Whenever there is a save version of an obscured runtime datum, use the save's.
- Resolution: `PlayerSaveData` is NOT a stable singleton (the game recreates it) -> live instance = the one with the MOST gold. `StageInfoData`/`LogManager` resolve by string. Singletons via the generic base `nn<a>` (`MonsterSpawnManager`, `LogManager`, **`ut`**): instance in the static of the generic, NOT by string. Pure 2-letter classes (uf/uo/xd/te/ud.su/yp) only by fixed offset from a readable class.
- Verified categories that are absent/already-covered: combo (does not exist — "combo" in the dump is the crafting UI); crit/element per hit (transient in DamageInfo); stamina/energy (does not exist — idle with no gate); prestige/ascension/rebirth (no class); elemental affinity (= per-element resistances, already in `Unit.bcfr`/`xd.bets`); achievements/missions (only MoreMountains/Steam cruft, not run quests); status effects (Chill/Freeze/Ignite/Shock/Bleed/Stun, 100% covered).
